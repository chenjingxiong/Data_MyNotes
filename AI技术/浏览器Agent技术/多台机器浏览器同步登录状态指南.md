## 概述

在多台机器间保持浏览器登录状态是日常工作和开发的常见需求。本文介绍多种方案，从简单的浏览器内置同步到高级的自托管解决方案。

| 方案 | 难度 | 安全性 | 灵活性 | 适用场景 |

| 浏览器内置同步 | ⭐ | ⭐⭐⭐⭐ | ⭐⭐ | 日常使用 |

## 方案一：浏览器内置同步

### Chrome / Edge 同步

#### 启用同步

1. **登录账号**
   - Chrome：登录 Google 账号
   - Edge：登录 Microsoft 账号

2. **开启同步设置**
   ```
   设置 → 您和 Google（或个人资料）→ 同步
   ```

3. **选择同步内容**
   - ✅ 已打开的标签页
   - ✅ 密码
   - ✅ 自动填充信息
   - ✅ 应用、扩展程序
   - ✅ 设置
   - ✅ 书签、历史记录

#### 高级设置

**仅同步特定数据类型**：
```
设置 → 同步和 Google 服务 → 加密同步数据
```

**手动触发同步**：
```
设置 → 您和 Google → 同步方式 → 立即同步
```

### Firefox 同步

#### 启用 Firefox Account

1. 打开 Firefox 账号页面：`about:preferences#sync`
2. 点击"登录 Firefox"
3. 注册或登录 Firefox 账号
4. 选择同步内容

#### 同步内容

- ✅ 书签
- ✅ 历史记录
- ✅ 标签页
- ✅ 密码
- ✅ 附加组件
- ✅ 偏好设置
- ✅ 地址栏

#### 自托管 Firefox Sync

如果需要完全控制数据：

```bash
# 使用 Docker 部署 Firefox Sync Server
docker run -d \
  --name firefox-sync \
  -p 5000:5000 \
  -v $(pwd)/data:/data \
  mozilla/syncserver:latest
```

配置 Firefox 指向自建服务器：
```
about:config → identity.sync.tokenserver.uri
→ https://your-server/token/1.0/sync/1.5
```

### Safari 同步（仅 macOS）

#### 启用 iCloud 同步

1. 系统设置 → Apple ID → iCloud
2. 开启"Safari"同步
3. 在所有 Apple 设备上使用相同 Apple ID

#### 跨平台限制

⚠️ **Safari 不支持跨平台同步**（仅限 Apple 生态系统）

## 方案二：Cookie 同步工具

### CookieCloud（推荐）

**详细介绍**：[[AI技术/Skills/CookieCloud 介绍与部署指南.md]]

#### 快速开始

1. **部署服务器**
   ```bash
   docker run -d \
     --name cookiecloud \
     -p 8088:8088 \
     -v $(pwd)/data:/data \
     --restart always \
     easychen/cookiecloud:latest
   ```

2. **安装插件**
   - Chrome: [Chrome Web Store](https://chrome.google.com/webstore/detail/cookiecloud/ffjiejobkoibkjlhjnlgmcnnigeelbdl)
   - Edge: [Edge Add-ons](https://microsoftedge.microsoft.com/addons/detail/cookiecloud/bffenpfpjikaeocaihdonmgnjjdpjkeo)

3. **配置同步**
   - 设置服务器地址：`http://your-server:8088`
   - 配置 UUID 和密码
   - 选择要同步的域名



#### 优势

- 🔒 端对端加密
- 🌐 自托管
- 📱 支持移动端
- ⚙️ 支持过滤域名
- 🔄 支持定时同步

### LocalStorage Bridge

适用于需要同步 LocalStorage 的场景：

```javascript
// 保存 LocalStorage
localStorage.setItem('user_token', 'xxx');

// 通过 CookieCloud 同步到其他设备
// 在其他设备上恢复
const token = localStorage.getItem('user_token');
```

## 方案三：密码管理器

### Bitwarden

#### 自部署 Bitwarden

```bash
# 使用 Docker 部署
docker run -d \
  --name bitwarden \
  -p 80:80 \
  -p 443:443 \
  -v $(pwd)/bw-data:/data \
  bitwarden/self-host:latest
```

或使用轻量级替代品 **Vaultwarden**：

```bash
docker run -d \
  --name vaultwarden \
  -p 80:80 \
  -v $(pwd)/vw-data:/data \
  vaultwarden/server:latest
```

#### 浏览器插件配置

1. 安装 Bitwarden 浏览器插件
2. 设置服务器 URL（自建版）
3. 登录账号
4. 启用自动填充

#### 使用场景

- 🏢 **团队协作**：共享团队账号
- 🔐 **高安全性要求**：支持 2FA
- 📱 **跨平台**：支持所有主流平台

### 1Password

团队协作推荐方案：

- ✅ 优秀的团队共享功能
- ✅ 旅行模式（清除敏感数据）
- ✅ Watchtower（密码健康检查）
- ❌ 不支持自部署

### KeePass + 云存储

本地优先方案：

1. 使用 KeePass/KeePassXC 管理密码
2. 将数据库文件存储在：
   - Nextcloud / ownCloud
   - Synology Drive
   - OneDrive / Google Drive
3. 使用浏览器插件自动填充

## 方案四：Session 共享工具

### Session Buddy

Chrome 扩展，保存和恢复浏览器会话：

1. 安装 [Session Buddy](https://chrome.google.com/webstore/detail/session-buddy/edacconmaakjimmfgnblocblbcdcpbko)
2. 保存当前会话
3. 导出为 JSON 文件
4. 在其他机器上导入

### Sessionbox

会话隔离和管理工具：

- 📦 每个 Tab 使用独立 Cookie
- 🔄 支持会话共享
- 👥 支持团队协作

### 手动导出 Cookie

适用于临时需求：

```javascript
// 在浏览器控制台执行
document.cookie.split(';').forEach(c => {
    console.log(c.trim());
});

// 或使用 EditThisCookie 插件导出
```

## 特殊场景方案

### 场景 1：开发环境同步

**问题**：开发和测试环境需要不同的登录状态

**解决方案**：

1. **使用 Chrome 多用户配置**
   ```bash
   # Windows
   "C:\Path\To\chrome.exe" --user-data-dir="C:\chrome-dev"

   # macOS
   /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --user-data-dir="/tmp/chrome-dev"

   # Linux
   google-chrome --user-data-dir="/tmp/chrome-dev"
   ```

2. **使用 Firefox 容器（Container Tabs）**
   - 安装 Multi-Account Containers 插件
   - 为每个环境创建独立容器
   - 容器间 Cookie 完全隔离

### 场景 2：团队账号共享

**问题**：团队成员需要共享同一账号（如测试账号）

**解决方案**：

1. **CookieCloud + 权限管理**
   - 部署团队专用 CookieCloud 服务器
   - 为每个服务配置不同 UUID
   - 定期轮换密码

2. **密码管理器团队版**
   - Bitwarden Teams
   - 1Password Business
   - 设置访问权限和审计日志

3. **专用浏览器配置**
   - 使用 Chrome/Edge 的企业策略
   - 预配置登录状态
   - 分发到团队成员

### 场景 3：自动化测试

**问题**：Playwright/Puppeteer 脚本需要登录状态

**解决方案**：

```javascript
// 使用 CookieCloud 获取 Cookie
const { chromium } = require('playwright');
const CryptoJS = require('crypto-js');

async function getCookies(uuid, password) {
    const response = await fetch(`http://server:8088/get/${uuid}`);
    const data = await response.json();

    const key = CryptoJS.MD5(uuid + '-' + password).toString().substring(0, 16);
    const decrypted = CryptoJS.AES.decrypt(data.encrypted, key).toString(CryptoJS.enc.Utf8);

    return JSON.parse(decrypted).cookie_data;
}

// 使用 Cookie 启动浏览器
const browser = await chromium.launch();
const context = await browser.newContext();
const cookies = await getCookies('uuid', 'password');
await context.addCookies(cookies);

const page = await context.newPage();
await page.goto('https://example.com');
// 已登录状态！
```

### 场景 4：移动端同步

**问题**：电脑浏览器登录状态同步到手机

**解决方案**：

1. **CookieCloud + Kiwi Browser**（Android）
   - 安装 Kiwi Browser（支持 Chrome 扩展）
   - 安装 CookieCloud 插件
   - 同步 Cookie

2. **Firefox Sync**（跨平台）
   - 桌面端和移动端都使用 Firefox
   - 登录相同 Firefox 账号
   - 自动同步登录状态

3. **快捷指令 + Shortcuts**（iOS）
   - 使用 iCloud 同步 Safari
   - 或使用 Firefox（iOS）

## 安全注意事项

### 🔒 数据加密

| 方案 | 加密方式 | 密钥控制 |
|------|----------|----------|
| Chrome/Edge 同步 | Google/Microsoft 服务器加密 | 厂商控制 |
| Firefox Sync | 端对端加密 | 用户控制 |
| CookieCloud | AES 端对端加密 | 用户控制 |
| Bitwarden | AES-256 加密 | 用户控制 |

### ⚠️ 安全最佳实践

1. **使用强密码**
   ```
   推荐：使用密码管理器生成的随机密码
   长度：至少 16 位
   复杂度：大小写 + 数字 + 符号
   ```

2. **启用 2FA/MFA**
   - Google 账号：2FA
   - Microsoft 账号：2FA
   - Firefox：2FA（可选）
   - 密码管理器：必须启用 2FA

3. **定期审查同步内容**
   ```
   建议每季度检查：
   - 同步的设备列表
   - 同步的密码列表
   - 异常登录记录
   ```

4. **敏感账号隔离**
   - 🔐 银行、支付账号：不同步
   - 🏢 企业账号：使用企业方案
   - 🧪 测试账号：可以同步

5. **设备安全**
   - 所有设备启用全盘加密
   - 使用强密码/生物识别
   - 及时更新系统和浏览器

### 🚫 不同步的内容

建议**手动管理**的敏感内容：

- 🔐 银行和金融账号
- 💳 支付和信用卡信息
- 🏥 医疗和健康记录
- 🏢 企业内部系统
- 🔑 加密货币私钥

## 常见问题

### Q1: 为什么同步后仍然需要重新登录？

**可能原因**：

1. **IP 地址变化**
   - 某些服务检测 IP 变化会要求重新验证
   - 解决：使用 VPN 保持相同 IP

2. **Cookie 属性限制**
   - `SameSite=Strict` 的 Cookie 不会被同步
   - `Secure` 属性的 Cookie 在 HTTP 下无效

3. **服务器时间不同步**
   - Cookie 时间戳验证失败
   - 解决：同步所有机器的系统时间

4. **指纹识别**
   - 浏览器指纹不同（Canvas、WebGL）
   - 解决：使用隐私模式或反指纹插件

### Q2: 多台电脑同时登录会被检测吗？

**答案**：取决于服务提供商

| 服务类型 | 策略 |
|---------|------|
| 大多数网站 | 允许多设备登录 |
| 银行/金融 | 通常单设备或需要验证 |
| 流媒体（Netflix） | 限制同时登录设备数 |
| 企业应用 | 通常强制单点登录 |

**规避方法**：

- 使用不同的浏览器配置文件
- 使用隐私/无痕模式
- 联系管理员增加设备配额

### Q3: 同步会泄露我的隐私吗？

**风险分析**：

| 方案 | 隐私风险 |
|------|----------|
| Chrome/Edge 同步 | 中（Google/Microsoft 可访问） |
| CookieCloud（自托管） | 低（端对端加密） |
| Firefox Sync | 低（端对端加密） |
| 密码管理器 | 低（端对端加密） |

**建议**：

- 敏感账号使用端对端加密方案
- 自托管服务（CookieCloud、Bitwarden）
- 定期审查同步内容

### Q4: 如何撤销已同步的登录状态？

**操作步骤**：

1. **修改密码**
   - 最直接有效的方法
   - 所有设备的旧 Cookie 将失效

2. **清除所有设备 Cookie**
   ```
   Chrome: 设置 → 隐私和安全 → 清除浏览数据 → Cookie
   ```

3. **撤销同步授权**
   - Google: https://myaccount.google.com/permissions
   - Microsoft: https://account.microsoft.com/permissions
   - Firefox: about:preferences#sync → 断开连接

4. **远程注销**
   - 访问网站的安全设置页面
   - 选择"注销所有设备"

### Q5: 企业环境如何管理？

**企业推荐方案**：

1. **SSO（单点登录）**
   - Okta、Azure AD、Auth0
   - 统一身份认证

2. **企业密码管理**
   - 1Password Business
   - Bitwarden Teams
   - LastPass Enterprise

3. **浏览器策略管理**
   - Chrome Enterprise
   - Microsoft Edge for Business
   - Firefox ESR

4. **零信任架构**
   - 设备信任评估
   - 持续认证
   - 最小权限原则
