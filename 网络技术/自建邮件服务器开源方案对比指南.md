# 自建邮件服务器开源方案对比指南

> [!info] 前言
> 基于 GitHub Stars 排名的自建邮件服务器对比，覆盖 **11 个高星开源项目**。重点关注 **低内存 / 低 CPU 占用**，并标注每个方案的 **随机账号 / Catch-all / 别名自动生成** 能力。

> [!warning] 自建邮件的前提条件
> 1. **IP 信誉**：VPS 的 IP 不在黑名单中 → 用 [MXToolbox](https://mxtoolbox.com/) 检查
> 2. **反向 DNS（rDNS/PTR）**：必须在 VPS 控制面板配置
> 3. **端口 25 开放**：部分云商（AWS、阿里云）默认封锁，需提前申请
> 4. **DNS 三件套**：SPF + DKIM + DMARC 必须正确配置

> [!note] "随机账号能力"说明
> - **Catch-all**：`*@yourdomain.com` 全部投递到一个邮箱，等效于无限随机地址
> - **Subaddressing**：`you+randomtag@domain.com` → `you@domain.com`
> - **随机别名生成**：一键生成 `a7x9k@domain.com` 等随机地址，可独立禁用/删除
> - 评分：✅✅ = 核心功能，✅ = 支持但需手动配置

### 1. Docker Mailserver (DMS)

| 项目 | 详情 |
|------|------|
| **GitHub** | [docker-mailserver/docker-mailserver](https://github.com/docker-mailserver/docker-mailserver) ★~15k |
| **语言** | Shell / Docker（Postfix + Dovecot） |
| **许可** | MIT |
| **架构** | 单容器（Postfix + Dovecot + Rspamd + ClamAV 等全打包） |

**功能亮点：**
- Postfix（MTA）+ Dovecot（IMAP/MDA）经典组合
- Rspamd / SpamAssassin / ClamAV 可选启用
- OpenDKIM + OpenDMARC + SPF
- Fail2Ban 防暴力破解
- Sieve 邮件过滤规则
- Let's Encrypt 自动证书
- LDAP / OAuth2 认证

**随机账号能力：**
- ✅ **Catch-all**：通过 Postfix `virtual_alias_maps` 配置
- ✅ **Subaddressing**（`+` 扩展）：Dovecot 原生支持 `user+tag@domain.com`
- ✅ **邮件别名**：`setup.sh alias add` 命令管理

**资源占用：**
```
空闲（无 ClamAV）:  ~200-300MB RAM
正常负载:           ~300-500MB RAM
开启 ClamAV:       +500MB RAM
CPU 空闲:           ~0-2%
```

> [!tip] 低资源优化
> 关闭 ClamAV 可节省 ~500MB RAM。对个人邮箱影响很小。

**优点：** 社区最活跃（15k Stars）、文档丰富、MIT 许可、基于工业级组件
**缺点：** 无 Web 管理面板（命令行管理）、初学者配置门槛较高
**适合：** 有 Docker 经验、追求可靠性的用户

### 6. SimpleLogin — 随机别名生成专家 ⭐推荐

| 项目         | 详情                                                             |
| ---------- | -------------------------------------------------------------- |
| **GitHub** | [simple-login/app](https://github.com/simple-login/app) ★~6.6k |
| **语言**     | Python（Flask）                                                  |
| **许可**     | MIT                                                            |
| **架构**     | Docker / 系统级安装                                                 |

**功能亮点：**
- 🎯 **核心功能就是随机别名生成**
- 一键创建随机邮箱别名（如 `epic-tiger-42@yourdomain.com`）
- 自定义别名（如 `netflix@yourdomain.com`）
- 每个别名可独立 **启用/禁用/删除**
- 别名转发到你的真实邮箱
- 浏览器扩展（Chrome / Firefox）
- 移动端 App（iOS / Android）
- PGP 加密支持
- 多域名支持

**随机账号能力（✅✅ 核心功能）：**
- ✅✅ **随机别名生成**：一键生成 `random-words@domain.com`
- ✅✅ **自定义别名**：指定前缀如 `shop-xxx@domain.com`
- ✅✅ **批量生成**：可批量创建多个别名
- ✅ **每个别名独立控制**：禁用垃圾邮件来源别名
- ✅ **别名活动日志**：查看每个别名的邮件流量

> [!important] 工作原理
> SimpleLogin 本身不是邮件服务器，而是**邮件别名/转发服务**。你需要配合一个实际的邮件服务器（如 Stalwart、DMS、Gmail 等）使用。
>
> 流程：`随机别名@yourdomain.com` → SimpleLogin 转发 → `你的真实邮箱`
>
> 优势：注册网站时用不同别名，一旦某个别名泄露垃圾邮件，直接禁用即可。

**资源占用：**
```
空闲:    ~150-256MB RAM（含 PostgreSQL）
正常:    ~256-512MB RAM
CPU:     ~0-2%
```

**优点：** 随机别名生成最成熟、MIT 许可、多端 App、隐私保护极佳
**缺点：** 不是完整邮件服务器（需配合 MTA/IMAP）、需单独部署数据库
**适合：** 已有邮件服务器、需要随机别名隐私保护的用户

**缺点：** 非传统邮箱方案（无 IMAP 收信）、资源占用较高
**适合：** 开发者 / SaaS 平台需要自建 transactional email 投递

## 随机账号能力专题对比

> [!abstract] 三种实现随机邮箱地址的方式

### 方式一：Catch-all（最简单）

任何发送到 `*@yourdomain.com` 的邮件都投递到你的邮箱。等效于拥有无限个随机地址。

| 方案 | 支持情况 | 配置难度 |
|------|----------|----------|
| Docker Mailserver | ✅ 支持 | 中（需编辑 Postfix 配置） |
| Mail-in-a-Box | ✅ 支持 | 低（Web 面板） |
| Mailcow | ✅ 支持 | 低（Web 面板） |
| Stalwart | ✅ 支持 | 低（配置文件） |
| mox | ✅ 支持 | 低（配置文件） |
| Maddy | ✅ 支持 | 中（路由规则配置） |
| Mailu | ✅ 支持 | 低（Web 面板） |

**缺点**：所有垃圾邮件都会进入你的邮箱，无法按地址独立禁用。

### 方式二：Subaddressing（`+` 扩展）

`yourname+anything@domain.com` → 投递到 `yourname@domain.com`。

| 方案 | 支持情况 | 说明 |
|------|----------|------|
| Docker Mailserver | ✅ 原生 | Dovecot 支持 |
| Mailcow | ✅ 原生 | 面板可配置 |
| Stalwart | ✅ 原生 | 原生支持 |
| Mail-in-a-Box | ✅ 原生 | Postfix 支持 |

**缺点**：部分网站不允许 `+` 符号注册；别人可以看出你的真实邮箱。

### 方式三：随机别名服务（最推荐） ⭐

通过 SimpleLogin 或 AnonAddy 生成完全独立的随机地址，每个地址可以单独禁用。

```
┌──────────────────────────────────────────────────────────┐
│  注册网站时                                              │
│                                                          │
│  SimpleLogin 生成: epic-tiger-42@yourdomain.com          │
│       │                                                  │
│       ▼                                                  │
│  SimpleLogin 转发 ──────► your-real@email.com            │
│                                                          │
│  如果 epic-tiger-42 被泄露/发垃圾邮件                     │
│       │                                                  │
│       ▼                                                  │
│  一键禁用该别名 ✂️  → 垃圾邮件消失                        │
└──────────────────────────────────────────────────────────┘
```

| 服务 | GitHub Stars | 语言 | 随机生成 | 独立禁用 | 移动端 | 许可 |
|------|-------------|------|----------|----------|--------|------|
| **SimpleLogin** | ~6.6k | Python | ✅ 随机词组 | ✅ | iOS + Android | MIT |
| **AnonAddy** | ~4.6k | PHP | ✅ UUID / 随机 | ✅ | Web only | MIT |

## 方案选择决策树

```
你的 VPS 内存是多少？
│
├─ ≤ 256MB（极低资源）
│  ├─ 只需要收发邮件 → Maddy（~40MB RAM）
│  └─ 需要管理面板 → mox（~60MB RAM）
│
├─ 256MB - 512MB（低资源）
│  ├─ 追求现代化 + 面板 → Stalwart（~128MB RAM）⭐
│  ├─ Docker 爱好者 → Docker Mailserver（关闭 ClamAV）
│  └─ 只需随机别名 → SimpleLogin / AnonAddy
│
├─ 512MB - 1GB（中等资源）
│  ├─ 需要面板 + Webmail → Mailu 或 Poste.io
│  ├─ 一键部署 → Mail-in-a-Box（需 Ubuntu 独立机）
│  └─ 邮件营销 → BillionMail
│
└─ 2GB+（资源充裕）
   ├─ 需要完整群件 → Mailcow
   └─ 大规模投递 → Postal
```

## 推荐组合方案

### 🏆 方案 A：极低资源 + 随机别名（推荐）

```
Stalwart (邮件服务器, ~128MB) + SimpleLogin (随机别名, ~256MB)
总计: ~384MB RAM
```

- Stalwart 处理邮件收发（SMTP + IMAP + JMAP）
- SimpleLogin 管理随机别名（注册网站时用）
- 合计仅需 512MB 以下的 VPS

### 🏆 方案 B：轻量全能

```
mox (邮件服务器, ~128MB)
总计: ~128MB RAM
```

- mox 自带 Webmail + 管理面板 + 反垃圾
- Catch-all 支持随机地址接收
- 最低只需 256MB VPS

### 🏆 方案 C：开箱即用

```
Mail-in-a-Box (邮件服务器, ~512MB)
总计: ~512MB RAM
```

- 一键部署，零配置
- Catch-all + 别名 + Nextcloud 群件
- 需要 1GB Ubuntu 独立 VPS

## DNS 必配记录

| 记录 | 作用 | 示例 |
|------|------|------|
| **MX** | 邮件服务器 | `@ MX 10 mail.yourdomain.com.` |
| **A** | 服务器 IP | `mail A 1.2.3.4` |
| **SPF** | 授权发送 | `@ TXT "v=spf1 mx ~all"` |
| **DKIM** | 邮件签名 | `selector._domainkey TXT "v=DKIM1; k=rsa; p=..."` |
| **DMARC** | 策略 | `_dmarc TXT "v=DMARC1; p=reject; rua=mailto:..."` |
| **PTR** | 反向 DNS | 在 VPS 控制面板配置 |
