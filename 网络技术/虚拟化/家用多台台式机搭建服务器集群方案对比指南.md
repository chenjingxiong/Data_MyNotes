# 家用多台台式机搭建服务器集群：候选方案与对比指南

> 利用家里多台闲置台式机，搭建属于自己的服务器集群——私有云、CI/CD 流水线、AI 推理平台、媒体服务器，一站搞定。

---

## 前置准备

### 硬件基础

| 项目 | 最低要求 | 建议配置 |
|------|----------|----------|
| CPU | 4 核 x86_64 | 8 核+，支持 VT-x/AMD-V |
| 内存 | 8 GB/台 | 16-32 GB/台 |
| 存储 | 256 GB SSD | 512 GB SSD + HDD 做数据盘 |
| 网络 | 千兆以太网 | 2.5G 或万兆（集群内部通信） |
| 台式机数量 | 2 台 | 3-5 台 |

### 网络拓扑建议

```
                    ┌─────────────┐
                    │  路由器/交换机  │
                    └──────┬──────┘
           ┌───────────────┼───────────────┐
           │               │               │
     ┌─────┴─────┐  ┌─────┴─────┐  ┌─────┴─────┐
     │  Node-01  │  │  Node-02  │  │  Node-03  │
     │ (Master)  │  │ (Worker)  │  │ (Worker)  │
     └───────────┘  └───────────┘  └───────────┘
```

**建议**：所有节点使用有线连接，分配静态 IP，主机名设为 `node-01`、`node-02` 等。

### 通用步骤（所有方案都需要）

1. **安装系统**：推荐 Debian 12/13 或 Ubuntu 24.04 LTS（Server 版）
2. **配置静态 IP**：在路由器 DHCP 中绑定 MAC 地址，或手动配置
3. **设置主机名**：`hostnamectl set-hostname node-01`
4. **配置 SSH 免密登录**：所有节点之间互通
5. **关闭 Swap**（K3s 方案需要）：`swapoff -a && sed -i '/swap/d' /etc/fstab`
6. **时间同步**：`apt install chrony && systemctl enable chrony`

## 方案一：Proxmox VE 虚拟化集群

### 简介

Proxmox VE 是基于 Debian 的开源虚拟化管理平台，支持 KVM 虚拟机和 LXC 容器，自带 Web 管理界面和集群功能。

### 架构

```
┌────────────────────────────────────────┐
│            Proxmox 集群                  │
│  ┌──────┐  ┌──────┐  ┌──────┐         │
│  │ VM-1 │  │ VM-2 │  │ CT-1 │  ...    │
│  └──────┘  └──────┘  └──────┘         │
│  ┌─────────────────────────────────┐   │
│  │     Proxmox VE + Ceph/ZFS       │   │
│  └─────────────────────────────────┘   │
│  ┌─────────────────────────────────┐   │
│  │       物理服务器 (Debian)         │   │
│  └─────────────────────────────────┘   │
└────────────────────────────────────────┘
```

### 安装步骤

1. 下载 Proxmox VE ISO（<https://www.proxmox.com/en/downloads>）
2. 每台机器独立安装 Proxmox VE（覆盖整个硬盘）
3. 通过 Web 界面（`https://node-ip:8006`）配置集群：
   ```bash
   # 在第一台节点上创建集群
   pvecm create my-cluster
   
   # 在其他节点上加入集群
   pvecm add node-01-ip
   ```

### 核心能力

- **KVM 虚拟机**：完整虚拟化，运行 Windows/Linux 任意系统
- **LXC 容器**：轻量级，共享宿主内核，性能接近原生
- **Ceph 分布式存储**：跨节点数据冗余和迁移
- **高可用（HA）**：虚拟机自动故障转移
- **Web 管理界面**：开箱即用的 GUI
- **备份与快照**：内置备份方案

### 适合场景

- 需要运行多种不同操作系统
- 需要完整的虚拟化（Windows、macOS 等）
- 想要图形化管理界面
- 需要高可用和虚拟机热迁移

### 优缺点

| 优点 | 缺点 |
|------|------|
| GUI 管理，上手简单 | 占用资源较多（虚拟化开销） |
| 支持 VM + 容器混合 | Ceph 配置复杂，至少 3 节点 |
| 高可用、热迁移 | 单独占盘，不能与其他系统共存 |
| 生态成熟，社区活跃 | 免费版无企业支持，订阅提醒 |

## 方案二：K3s 轻量级 Kubernetes 集群

### 简介

K3s 是 Rancher 出品的轻量级 Kubernetes 发行版，单二进制文件，内存占用低，适合家庭实验室和边缘计算。

### 架构

```
┌──────────────────────────────────────────────┐
│              K3s Kubernetes 集群               │
│                                              │
│  ┌───────────┐  ┌───────────┐  ┌──────────┐ │
│  │  Master    │  │  Worker   │  │  Worker  │ │
│  │  (Server)  │  │  (Agent)  │  │  (Agent) │ │
│  │            │  │           │  │          │ │
│  │  ┌──────┐  │  │  ┌──────┐│  │  ┌─────┐ │ │
│  │  │ Pod  │  │  │  │ Pod  ││  │  │ Pod │ │ │
│  │  └──────┘  │  │  └──────┘│  │  └─────┘ │ │
│  └───────────┘  └───────────┘  └──────────┘ │
│                                              │
│  ┌──────────────────────────────────────────┐│
│  │  Containerd + Flannel/VxLAN + SQLite     ││
│  └──────────────────────────────────────────┘│
└──────────────────────────────────────────────┘
```

### 安装步骤

```bash
# Master 节点
curl -sfL https://get.k3s.io | sh -s - server --cluster-init

# 获取 Token
cat /var/lib/rancher/k3s/server/node-token

# Worker 节点
curl -sfL https://get.k3s.io | K3S_URL=https://master-ip:6443 K3S_TOKEN=<token> sh -
```

一条命令即可完成节点加入，自动配置网络和证书。

### 核心能力

- **Pod 调度**：自动在节点间分配工作负载
- **Service 发现**：内置 DNS 和负载均衡
- ** Helm Chart**：一键部署复杂应用
- **Ingress**：统一入口管理
- **持久化**：支持 Local Path Provisioner、Longhorn 等
- **自动更新**：Rancher System Upgrade Controller

### 推荐部署的应用

| 应用                   | 说明                  | Helm Chart              |
| -------------------- | ------------------- | ----------------------- |
| Longhorn             | 分布式块存储              | `longhorn/longhorn`     |
| Traefik              | Ingress 控制器（K3s 自带） | 内置                      |
| Prometheus + Grafana | 监控告警                | `kube-prometheus-stack` |
| Vaultwarden          | 密码管理                | 自定义                     |
| Nextcloud            | 私有云盘                | `nextcloud/nextcloud`   |
| Jellyfin             | 媒体服务器               | 自定义                     |
| Gitea                | 轻量 Git 服务           | `gitea-charts/gitea`    |

### 适合场景

- 主要运行容器化应用
- 想学习 Kubernetes 生态
- 需要自动调度和弹性伸缩
- CI/CD 流水线、微服务架构

### 优缺点

| 优点 | 缺点 |
|------|------|
| 轻量，512MB 即可运行 | 学习曲线陡峭（K8s 生态） |
| 云原生标准，技能可迁移 | 排错复杂，YAML 地狱 |
| 生态丰富，Helm Chart 海量 | 不适合非容器化工作负载 |
| 社区庞大，文档齐全 | 调试网络问题困难 |

## 方案三：Docker Swarm 集群

### 简介

Docker 内置的容器编排工具，最简单的集群方案，适合小规模部署。

### 架构

```
┌─────────────────────────────────────┐
│         Docker Swarm 集群            │
│                                     │
│  ┌──────────┐  ┌──────────┐        │
│  │ Manager   │  │ Worker   │        │
│  │ ┌──────┐  │  │ ┌──────┐ │        │
│  │ │Container│ │  │ │Container││        │
│  │ └──────┘  │  │ └──────┘ │        │
│  └──────────┘  └──────────┘        │
│                                     │
│  ┌─────────────────────────────────┐│
│  │  Docker Engine + Overlay Network ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
```

### 安装步骤

```bash
# Manager 节点
docker swarm init --advertise-addr <manager-ip>

# 获取加入命令
docker swarm join-token worker

# Worker 节点执行输出的命令
docker swarm join --token <token> <manager-ip>:2377
```

### 核心能力

- **Service 部署**：声明式服务管理
- **负载均衡**：内置 VIP 和 DNSRR
- **滚动更新**：零停机部署
- **Overlay 网络**：跨节点容器通信
- **Secret/Config**：配置和密钥管理

### 适合场景

- 只需简单的容器编排
- 团队已熟悉 Docker
- 不需要 K8s 的复杂功能
- 快速搭建，维护成本低

### 优缺点

| 优点 | 缺点 |
|------|------|
| 上手最快，5 分钟建集群 | 功能有限，生态萎缩 |
| Docker 原生，无需额外组件 | 无自动调度（只按 replica 分配） |
| 配置简单，YAML 少 | 社区活跃度下降 |
| 维护成本最低 | 缺少高级功能（HPA、CRD 等） |

## 方案四：HashiCorp Nomad 集群

### 简介

HashiCorp 出品的工作负载编排器，支持容器和非容器化应用，与 Consul、Vault 生态深度集成。

### 架构

```
┌──────────────────────────────────────────┐
│           Nomad 集群                      │
│                                          │
│  ┌──────────┐  ┌──────────┐  ┌────────┐ │
│  │ Server   │  │ Client   │  │ Client │ │
│  │ (Raft)   │  │ (Driver) │  │(Driver)│ │
│  │          │  │ ┌──────┐ │  │┌──────┐│ │
│  │          │  │ │ Task │ │  ││ Task ││ │
│  │          │  │ └──────┘ │  │└──────┘│ │
│  └──────────┘  └──────────┘  └────────┘ │
│                                          │
│  Consul (服务发现) + Vault (密钥管理)       │
└──────────────────────────────────────────┘
```

### 安装步骤

```bash
# 所有节点安装 Nomad
wget https://releases.hashicorp.com/nomad/1.9.6/nomad_1.9.6_linux_amd64.zip
unzip nomad_1.9.6_linux_amd64.zip -d /usr/local/bin/

# Server 节点配置 /etc/nomad.d/server.hcl
# Client 节点配置 /etc/nomad.d/client.hcl

# 启动
systemctl enable --now nomad
```

### 核心能力

- **多运行时**：Docker 容器 + 原生进程 + Java + QEMU 虚拟机
- **调度器**：binpack/spread 等多种调度策略
- **Consul 集成**：服务发现和健康检查
- **Vault 集成**：密钥和证书自动轮转
- **联邦集群**：跨数据中心编排

### 适合场景

- 混合工作负载（容器 + 原生进程 + 虚拟机）
- 已使用 Consul/Vault 生态
- 需要简单但灵活的编排
- 不想学 K8s 的复杂性

### 优缺点

| 优点 | 缺点 |
|------|------|
| 单二进制，架构简洁 | 社区远小于 K8s |
| 支持非容器工作负载 | 生态工具链少 |
| 与 HashiStack 深度集成 | 学习资源有限 |
| 调度灵活，配置直观 | 企业版功能需付费 |

## 方案五：Proxmox + K3s 混合方案（推荐）

### 简介

在 Proxmox 上虚拟出 K3s 节点，兼顾虚拟化和容器编排的优势——最灵活但资源开销最大。

### 架构

```
┌──────────────────────────────────────────────────────┐
│                    物理服务器 × N                       │
│                                                      │
│  ┌────────────────────────────────────────────────┐  │
│  │  Proxmox VE 集群                                │  │
│  │                                                │  │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────┐    │  │
│  │  │ K3s-Master│  │ K3s-Worker│  │ 其他 VM  │    │  │
│  │  │   (VM)    │  │   (VM)    │  │ (Win/NAS)│    │  │
│  │  └──────────┘  └──────────┘  └──────────┘    │  │
│  │                                                │  │
│  │  ┌──────────┐  ┌──────────┐                   │  │
│  │  │ LXC 容器  │  │ LXC 容器  │  ...             │  │
│  │  └──────────┘  └──────────┘                   │  │
│  └────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────┘
```

### 部署思路

1. 每台物理机安装 Proxmox VE
2. 在 Proxmox 上创建 K3s VM（2C4G 起步）或 LXC 容器运行 K3s
3. 同时运行其他 VM（Windows、NAS 等）
4. 使用 Ceph 或 ZFS 做共享存储
5. K3s 负责 Docker 化服务，Proxmox 负责系统级虚拟化

### 适合场景

- 硬件配置较高（16GB+ 内存/台）
- 需要同时运行容器和虚拟机
- 想要最大灵活性

### 优缺点

| 优点 | 缺点 |
|------|------|
| 灵活度最高 | 资源开销最大（双重虚拟化） |
| 兼顾 VM 和容器 | 运维复杂度最高 |
| 可以运行任意系统 | 故障排查链路更长 |
| 可逐步演进 | 需要更多内存和存储 |

## 常见问题

### Q：家里带宽有限，会影响集群吗？

集群内部通信走局域网，不经过外网。只有对外暴露的服务才需要上传带宽。建议所有节点使用有线千兆网络。

### Q：节点配置不一致怎么办？

K3s 和 Nomad 支持标签/污点调度，可以把大负载调度到高配节点。Proxmox 支持资源限额分配。

### Q：断电怎么办？

- 设置 BIOS 来电自动开机（AC Power Recovery → Power On）
- Proxmox HA 可以自动恢复
- K3s Pod 会自动重新调度
- 重要数据务必做冗余（Ceph/ZFS RAID）

### Q：需要多少电费？

单台台式机功耗约 50-150W，3 台 24 小时运行约 200-400W，月电费约 100-300 元（按 0.6 元/度计算）。建议在 BIOS 中开启节能模式，关闭独显使用核显。
